03 Data Model
Entity Types (nodes)
workspace
Root registration unit. Maps to a root path on disk.
repository
VCS context (git or none). Holds branch state.
package
Manifest-backed unit (package.json, .csproj).
file
Indexed file with language, summary, and hash.
module
Canonical code module. Has exports, dep counts.
symbol
Function, class, type, enum — with range + signature.
commit
Git commit. SHA, author, summary, branch.
change_group
Set of files changed in a commit.
test_suite / test_case
Detected test files and individual cases.
architecture_violation
Persisted rule breach with severity + detail.
log_event
Ingested log entry with level, service, message.
incident
Correlated failure event for incident timelines.
Edge Types (relationships)

04 Indexing Pipeline
Phase 1
workspace-manager

Register or retrieve workspace. Resolve app paths under ~/.local-engineering-brain/. Auto-detect workspace from CLI args, env vars, or CWD markers (CLAUDE_PROJECT_DIR, CODEX_WORKSPACE_ROOT, etc.).

workspace-manager storage-sqlite shared-utils
Phase 2
discovery-engine

Glob all files. Apply .gitignore rules. Classify each candidate as source / test / config / manifest / doc. Skip node_modules/, dist/, coverage/. Build candidate file list for parsing.

discovery-engine language-core shared-utils
Phase 3
language adapters

Route each file to the matching language adapter. TypeScript/JS: deep AST parse — extracts symbols, imports, exports, call edges. C# / Lua: guaranteed-discovery mode — file + manifest evidence. Unrecognized files: file-level indexing only.

language-ts language-csharp language-lua language-core
Phase 4
intel collectors

Parallel collection of supplementary intelligence: git-intel walks branch commits and changed files; manifest-intel extracts package manager hints; test-intel identifies test frameworks and cases; log-intel ingests workspace logs.

git-intel manifest-intel test-intel log-intel
Phase 5
architecture-rules

Evaluate user-configured allow/forbid rules against the assembled edge graph. Emit architecture_violation entities for breaches. Violations are persisted and queryable — not re-evaluated per request.

architecture-rules storage-sqlite
Phase 6
storage-sqlite

Persist all entities and edges via typed upsert. SQLite WAL mode for concurrent reads during indexing. Stable IDs (sha256-based) ensure idempotent re-indexing. Single local database at ~/.local-engineering-brain/brain.db.

storage-sqlite core-types shared-utils
